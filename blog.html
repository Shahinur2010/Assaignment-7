Answers of the Required Questions

1. Difference between map, forEach, find, and filter

map() → Executes a function on each element of an array. Every element of the array is passed to the callback function and returns a new array with the same length.

forEach() →It is just a plain form of for loop that can be used to iterate through array items, it allows to change elements of original array and returns undefined.

The JavaScript array method find() → returns the first matching element.

While the array method filter() → returns an array of all matching elements.

2. Difference between Local Storage, Session Storage and Cookies

The local storage object stores data with no expiration date. The data is not deleted when the browser is closed, and are available for future sessions.

The session storage data is only available during the browser session. It stores data for one session (data is lost when the browser tab is closed).

Cookies in JavaScript are used to read from and store small data related to the user on the browser. Cookies are mainly for reading server-side.

3. Event Loop and it's functionality

The event loop is the secret behind JavaScript’s asynchronous programming. JS executes all operations on a single thread, but using a few smart data structures, it gives us the illusion of multi-threading.

Event loop in JavaScript is a mechanism through which the 'calls waiting for execution' in the callback queue/job queue can be put on the call stack.

The Event Loop has one simple job - to monitor the Call Stack and the Callback Queue. If the Call Stack is empty, the Event Loop will take the first event from the queue and will push it to the Call Stack, which effectively runs it. Such an iteration is called a tick in the Event Loop.
